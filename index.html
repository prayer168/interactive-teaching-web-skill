Retrospective

本次建置複盤

有效做法

  • 先確認年級、使用情境、是否需要數據輸入與 GitHub Pages。
  • 用分頁式 RWD 結構承載觀察、實驗、圖表、原理、生活應用與測驗。
  • 以 JavaScript canvas 取代靜態圖片，讓科學現象能被動態觀察。
  • 每次小修正後立即做文字搜尋、語法檢查與線上驗證。

主要問題

  • 同一專案可能同時存在工作區與指定 Google Drive 資料夾，容易改錯檔案。
  • 隱藏分頁內的 canvas 可能因尺寸尚未確定而空白。
  • 「不要捲軸」必須用實際 viewport 測量，不能只靠目測。
  • GitHub Pages 與社群連結預覽會快取，需要 cache-busting 與 metadata 搜尋。

流程改進

  • 開工先鎖定 authoritative folder，所有 patch、測試、git 操作都在同一資料夾。
  • 把可重用規則寫進 skill，把詳細複盤放到技術文件網頁。
  • 將檢查項目固定化：語法、文字、RWD、canvas、互動、部署。
  • 每次推送後用 query string 驗證 HTML、CSS、JS 都是新版。

Workflow

高效率建置流程

1

需求盤點

確認年級、單元、學生使用方式、教學來源、互動需求、指定資料夾、GitHub 帳號與 repository 名稱。

2

教學設計 brief

先提出目標、分頁架構、核心操作、資料圖表、RWD 策略與投影使用方式，再取得確認。

3

靜態網站實作

以 `index.html`、`styles.css`、`script.js`、`README.md`、`.nojekyll` 建置，除非明確需要框架。

4

互動與動畫

數值條件使用 slider，對照實驗並排呈現，生活應用採左側工具列與右側大型 canvas/文字說明。

5

驗證與修正

檢查語法、舊文字、單位、RWD、canvas 首幀、互動流程、圖表結果與無捲軸需求。

6

發布與收工

提交、推送、確認 GitHub Pages，最後把本次可重用經驗更新回 skill。

Architecture

建議網站架構

觀察站

導入生活現象與探究問題，提供學生可理解的第一個觀察任務。

互動實驗室

使用 slider 調整條件，並排比較結果，動畫逐步上升或變化。

我的圖表

學生輸入自己的實驗數據，產生圖表、最大值判讀與結論句。

原理小教室

用 canvas 小動畫呈現粒子、孔隙、吸附或水分移動等概念。

生活中的應用

至少六個例子，左側選單加圖示，右側提供大型動畫與短說明。

挑戰測驗

題數依需求調整，回饋要能連回實驗變因與生活應用。

Deliverables

交付物與維護規則

標準交付檔案

  • `index.html`：頁面結構、SEO 與分享 metadata。
  • `styles.css`：RWD、分頁狀態、無捲軸壓縮規則。
  • `script.js`：互動、圖表、canvas、測驗與狀態更新。
  • `README.md`：教材用途、部署與維護摘要。
  • `.nojekyll`：GitHub Pages 靜態站設定。

維護原則

  • 每次改動都先確認是否需要同步 HTML、CSS、JS、README 與 metadata。
  • 單位、年級、標題與公開分享文字必須一起搜尋替換。
  • 移除功能時，同步移除 dead HTML/CSS/JS。
  • 部署後等 Pages 更新，再驗證線上實際內容。
