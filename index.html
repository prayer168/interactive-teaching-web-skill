Overview

專案總覽

教材定位

  • 對象：國小四年級自然領域學生。
  • 主題：毛細現象的觀察、探究、資料判讀與生活應用。
  • 使用方式：學生自主操作為主，也可供教師投影示範。
  • 型態：RWD 靜態網站，部署到 GitHub Pages。

核心功能

  • 互動實驗室：用 slider 調整層數、寬度、長度與觀察時間。
  • 我的圖表：學生輸入實驗數據並產生長條圖與結論句。
  • 原理小教室：用 canvas 動畫呈現水分移動、孔隙與吸附概念。
  • 生活中的應用：六個生活案例，左側選單、右側動畫與說明。

技術選擇

  • HTML/CSS/JavaScript：維持低維護成本與 GitHub Pages 相容性。
  • Canvas：取代靜態圖片，表現水上升、滲透與擴散過程。
  • CSS active-tab 狀態：針對單一頁面做無捲軸版面壓縮。
  • GitHub Pages：公開分享、版本管理與快速更新。

Workflow

實際製作流程

1

需求確認

先確認年級、使用情境、資料輸入需求、網站風格、指定資料夾與 GitHub repo。這一步決定教材不是單純展示頁，而是學生可操作、可輸入數據、可看到動畫回饋的探究工具。

2

教學資料讀取

參考教學影片、逐字稿與既有範例網站，抽取適合國小學生的概念順序：觀察現象、提出預測、操作變因、記錄數據、解釋結果、連結生活。

3

分頁式資訊架構

建立「觀察站、互動實驗室、我的圖表、原理小教室、生活中的應用、挑戰測驗」等分頁。後續依教師回饋移除不需要的任務或紀錄單，保持畫面簡潔。

4

互動與動畫實作

把紙材比較改為同頁並排顯示，數值條件改成 slider。水位與吸水高度不是瞬間跳到結果，而是 6 到 8 秒逐步上升，讓學生看見過程。

5

資料圖表與測驗

將學生輸入的實驗數據轉為長條圖，單位統一為 cm。測驗題從六題擴充到八題，讓題目連結紙材、層數、寬度、毛細現象原理與生活應用。

6

部署與流程沉澱

發布到 GitHub Pages 後，持續依畫面截圖與實際瀏覽器狀態修正。最後把製作經驗整理成 `SKILL.md` 與這份技術文件，方便下次教材建置直接套用。

Architecture

網站架構與實作重點

觀察站

用生活情境引入問題，讓學生先想「水為什麼會往上跑」。畫面重點是短句、清楚圖像與可立即進入實驗。

互動實驗室

核心模擬區。衛生紙與廚房紙巾並排比較，條件使用 slider，結果以動畫逐步上升並同步更新 cm 數值。

我的圖表

學生輸入各材料的吸水高度，系統產生長條圖、最大值判讀與提示語。圖表標籤與結論句統一使用 cm。

原理小教室

三個概念區塊加入 canvas 動畫，避免只用文字說明。每個 canvas 都需要首幀、分頁切換 redraw 與 resize redraw。

生活中的應用

至少六個例子：抹布吸水、植物輸水、紙巾擴散、蠟燭燈芯、土壤保水、吸管花實驗。左側選單加入 Emoji 並置中。

挑戰測驗

八題測驗檢查概念理解。回饋文字要明確指出原因，例如紙材孔隙、寬度、層數或時間如何影響水移動。

Implementation Decisions

關鍵實作決策

為什麼不用大量圖片

  • 本教材需要呈現「過程」，靜態圖無法表達水逐步上升。
  • Canvas 可以依學生調整的參數即時改變高度、速度與圖像。
  • 減少外部圖片依賴，部署到 GitHub Pages 時更穩定。

為什麼用一頁式實驗室

  • 教師投影與學生平板操作時，捲動會打斷觀察流程。
  • 把控制區與結果區同時放在畫面內，學生能直接比較變因與結果。
  • 使用 active-tab CSS 只壓縮實驗室，不影響其他頁的閱讀性。

Pitfalls

踩過的坑與解法

01

改錯資料夾

問題：工作區與 Google Drive 指定資料夾都有相似檔案，若沒有先確認 authoritative folder，很容易修改到非正式版本。

解法：每次開工先查 `git status`、remote URL、檔案時間與教師指定路徑。所有 patch、測試、commit、push 都固定在正確資料夾。

02

Canvas 空白

問題：Canvas 放在隱藏分頁時，尺寸可能尚未穩定，切換頁面後看起來沒有動畫。

解法：先畫首幀，再啟動動畫；分頁切換與 resize 後重新讀取尺寸並 redraw，必要時延遲補繪一次。

03

無捲軸版面仍然超出

問題：看起來已經壓縮，但在特定瀏覽器高度下底部仍超出。

解法：用實際 viewport 量測 `scrollHeight` 與 `innerHeight`。只針對實驗室 active tab 調整 gap、padding、杯子高度與左右比例。

04

單位與年級殘留

問題：mL 改成 cm、三年級改成四年級時，容易漏掉圖表、測驗、README 或分享 metadata。

解法：針對舊單位、舊年級、舊標題做全域搜尋，HTML、CSS、JS、README、Open Graph 與 Twitter metadata 都要檢查。

05

GitHub Pages 快取

問題：push 成功不等於 Pages 立即更新，社群預覽也可能顯示舊資訊。

解法：用 `?v=commit` 驗證線上 HTML/CSS/JS 是否已更新。分享資訊要同步更新 meta tags，並提醒第三方平台可能需要等待。

06

Git 推送衝突

問題：同一 repo 從不同位置推送，遠端已有新提交時會 push rejected。

解法：先 fetch 或 pull --rebase，確認差異後再推送。不要用 reset 覆蓋教師資料或遠端版本。

標準交付內容

  • 每次收工要把可重用經驗更新回 skill，讓下一份教材更快完成。
